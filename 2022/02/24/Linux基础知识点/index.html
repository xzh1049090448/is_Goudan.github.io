常用命令

命令 执行操作
history 查看历史命令
ls 查看文件目录 -l 显示详细信息 -a 显示隐藏目录 -R递归显示子目录信息
cd 更改目录
pwd 查看当前的工作目录
mkdir 创建目录
tree 目录成树状显示
rmdir 删除目录，只能删非空目录
which 显示对应命令所在的路径
touch 创建文件，如果不存在创建，如果存在则只更改访问时间
rm 删除文件或者目录 -r 递归删除子目录 rm -rf * 强制删除目录下所有内容
cp 拷贝文件或者目录
mv 移动文件或者目录
cat 直接显示文件信息
more和less 分屏显示文件信息
head head -n file 显示文件头n行，不加-n默认10行
tail tail -n file 显示文件后n行；tail -f file 查看文件尾部内容，不退出，用于追踪日志
du -h 查看文件占用空间
ln 建立硬连接 ln hello hello.hard1 见下图 ； ln -s hello hello.hard3 建立软连接
unlink 删除软硬连接
chmod 改变文件权限
chown 改变文件所属权限 sudo chown 用户：组 文件名|目录
find find dir [opt] 内容 opt：-name -type -size 按名字、类型、大小找文件
grep 过滤信息
xargs 将前一目录的输出分成小块输入给后面的命令
zip zip -r zipname.zip filename1 filename2 … 压缩文件 -r递归子目录
unzip unzip zipname.zip 解压文件
tar tar zcvf zipname.tar.gz filename1 filename2 … 压缩文件
tar zxvf zipname.tar.gz 解压文件
rar rar a -r rarname（可以无后缀） filename1 filename2 … 压缩文件
rar x rarname.rar 解压文件

GCC流程

  • 示例代码
  • 预编译过程

这个过程处理宏定义和include，去除注释，不会对语法进行检查。

  • 编译过程

这个阶段，检查语法，生成汇编代码。

  • 汇编过程

这个阶段，生成目标代码。

此过程生成ELF格式的目标代码。

  • 链接过程

链接过程。生成可执行代码。链接分为两种，一种是静态链接，另外一种是动态链接。使用静态链接的好处是，依赖的动态链接库较少，对动态链接库的版本不会很敏感，具有较好的兼容性；缺点是生成的程序比较大。使用动态链接的好处是，生成的程序比较小，占用较少的内存。

动态库和静态库制作流程：https://blog.csdn.net/weixin_44718794/article/details/106599417

gdb调试

  • 启动gdb:gdb appapp为对应程序名。

  • 在gdb启动程序：

    • r(un) 启动。
    • start 启动-停留在 main函数，分步调试。
    • n(ext) 下一条指令。
    • s(tep) 下一条指令，可以进入函数内部，库函数不能进。
    • q(uit) 退出。
    • b(reak) n 在主函数所在文件 第n行设置断点， b func 在func函数设置断点， b main.cpp:2=10 在某个文件某一行设置断点；b main.cpp:2=10 if i==0设置条件断点。
    • l(ist) 查看主函数对应文件，list func.cpp:1看func从第一行 。
    • info b 查看断点。
    • d(el) n 删除断点，n对应info b里的第几个 断点。
    • c(ontinue) 跳到下一断点。
    • p(rint) context 打印context的内容。
    • ptype x 打印x类型。
    • set 设置变量值。
    • display argc 执行任何一个指令argc的值都显示一下。
    • undisplay n不显示display，n代表display的那个行号，也可以通过info display查看。

  • gdb追踪core

    • 查看core文件大小 ：ulimit -c
    • 设置生成core文件大小：ulimit -c unlimited 设置无限大
    • gdb app core
      • where 查看错误在哪。
